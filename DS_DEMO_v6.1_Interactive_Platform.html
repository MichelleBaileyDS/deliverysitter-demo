🚀 GitHub Pages Setup — 5 Minute Guide
One-time setup
💡
Once set up, your demo will be live at a permanent URL. Share that link + access code with anyone — they click, enter code, see only their role.
1
Create a free GitHub accountGo to github.com → Sign up → Use any username (e.g. deliverysitter-michelle)
2
Create a new repositoryClick the green "New" button → Name it: deliverysitter-demo → Tick "Public" → Click "Create repository"
3
Upload this fileOn your new repository page → Click "uploading an existing file" → Drag and drop this HTML file → Rename it to: index.html → Click "Commit changes"
4
Enable GitHub PagesIn your repository → Click "Settings" (top tab) → Left sidebar: "Pages" → Under "Source" select "main" branch → Click Save
5
Your live URL is ready in ~60 secondsGitHub will show your URL — it will look like:
https://deliverysitter-michelle.github.io/deliverysitter-demo/
6
Share with stakeholders — two options:Option A — Send the URL + their code separately (most secure)
Option B — Send a direct link with code embedded:
https://your-username.github.io/deliverysitter-demo/?access=DS-T4-INV-0001
7
Save to OneDrive as backupSave this HTML file to OneDrive → You can open it locally on your laptop for meetings too. Both work — same file, same codes.
